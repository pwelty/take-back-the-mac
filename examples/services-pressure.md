# Example: Services Pressure

Apple Services can be useful. The problem begins when the device experience bends around recurring revenue.

Patterns worth documenting:

- storage upgrade nudges
- subscription prompts
- service bundles promoted inside system experiences
- sign-in requirements for local-feeling features
- defaults that favor Apple services over user choice
- app-store rules that push payments, subscriptions, or discovery through Apple-owned rails

The principle is simple: Services should support the device experience. They should not become the device experience.
