# Contributing

Good contributions do one of four things:

- Improve the wording of a user right.
- Add a specific example with evidence.
- Propose an Apple-facing remedy.
- Document a repeatable pattern of user harm.

Please avoid generic Apple-bashing. The point is not rage for rage's sake. The point is a better bargain between Apple and the people who paid for the machines.

## Evidence Helps

Strong examples include:

- screenshots
- exact file paths
- package receipts
- launch item names
- app review examples
- subscription dark patterns
- steps to reproduce
- links to public documentation or reporting

## Tone

Be direct. Be specific. Be cranky if the crankiness has receipts.

Do not make claims you cannot defend.
